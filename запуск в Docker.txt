Шаг 1: Написание кода
Вы пишете или изменяете код ваших тестов в IntelliJ IDEA, как и раньше.

Шаг 2: Запуск всей системы одной командой
Открываете терминал в корневой директории проекта.
Выполняете команду:
docker-compose up --build

docker-compose up: Запускает все, что описано в docker-compose.yml (приложение, WireMock, тесты).
--build: Пересобирает образы, если вы внесли изменения в код или Dockerfile. Всегда используйте этот флаг, когда меняете код.

Шаг 3: Наблюдение за процессом
В терминале вы увидите логи от всех сервисов в реальном времени.
Сначала запустится wiremock, затем my-app. Вы должны увидеть знакомую строку Started AqaApplication....
Сразу после этого запустится контейнер tests, и в логах пойдет вывод Maven: компиляция и выполнение ваших тестов.
В конце вы увидите, что контейнер tests остановился (tests exited with code 0 — успех, code 1 — неудача).

Шаг 4: Просмотр отчета Allure
После завершения тестов в вашем проекте обновится папка target/allure-results.
Выполняете в терминале команду:
allure serve target/allure-results
Отчет автоматически откроется в браузере.

Шаг 5: Остановка и очистка
Когда вы закончили работу, нажмите в терминале, где запущен docker-compose, Ctrl+C.
Для полной остановки и удаления контейнеров (чтобы освободить ресурсы) выполните:
docker-compose down

Как это работает в CI/CD (на примере GitHub Actions)

Шаг 1: Триггер
Вы делаете git push в ветку main на GitHub.

Шаг 2: Запуск пайплайна
GitHub Actions автоматически обнаруживает ваш файл .github/workflows/ci.yml.
GitHub выделяет виртуальную машину (с Ubuntu) и начинает выполнять шаги из этого файла.

Шаг 3: Выполнение шагов
Checkout code: Виртуальная машина скачивает ваш код из репозитория.
Run tests with Docker Compose: Выполняется та же самая магия: docker-compose up --build.
Ключевой момент здесь — флаг --exit-code-from tests. Он говорит CI-системе: "Статус всего этого шага равен статусу завершения контейнера tests".
Если тесты прошли (code 0), шаг считается успешным (зеленая галочка).
Если тесты упали (code 1), шаг проваливается (красный крестик), и весь пайплайн останавливается с ошибкой.

Шаг 4: Результат
Вы заходите на страницу вашего проекта в GitHub на вкладку "Actions" и видите результат прогона — зеленый или красный.
